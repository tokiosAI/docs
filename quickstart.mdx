---
title: "Connect your local model to Tokios"
sidebarTitle: "Quickstart"
description: "Pair a connector, register your model, and try it — no inbound ports, public IP, or NAT changes."
---

Tokios reaches your private model over an outbound-only tunnel — nothing on your machine listens for inbound traffic. The console's **[Setup wizard](https://tokios.com/console/setup)** pairs a connector, approves it, and registers your model. From there you can try it instantly in the **Playground**, then create an API key and point your coding client at it.

Sign in at [tokios.com/console](https://tokios.com/console) and open the **Setup** tab. (First time? Confirm your email from the verification link — it's valid for 24 hours.)

<Steps>
  <Step title="Pair a connector">
    On the **Setup** tab, optionally name the connector, then click **Start pairing**. Tokios shows a one-time **claim code** like `TKS-6KZY-4ZAW`, valid for about 15 minutes.

    The claim code is a pairing handle — it is **not** your API key (you create that below).
  </Step>

  <Step title="Install the connector and approve it">
    On the machine next to your model, run the install one-liner the console shows for your OS. On Windows (PowerShell):

    ```powershell Windows
    irm https://tokios.com/downloads/install/windows | iex; tokios-install -Code 'TKS-6KZY-4ZAW'
    ```

    Already have `tokios-connector` installed? Skip the installer and pass the claim code directly:

    ```bash Already installed
    tokios-connector TKS-6KZY-4ZAW
    ```

    The connector dials out over a single WebSocket — no inbound ports, no public IP. Back in the console, **approve** the device. Your token is delivered securely over the tunnel; you never paste it into a file.
  </Step>

  <Step title="Register your model">
    In the [Models](https://tokios.com/console/models) tab, pick the model to expose and the **public name** your clients will use — for example, `gemma-tunnel`. That public name is the `"model"` value in every API request.
  </Step>

  <Step title="Try it in the Playground">
    Open the [Playground](https://tokios.com/console/playground) tab, pick your model, type a prompt, and click **Run**. It authenticates with your current session — no API key needed — so you can confirm the tunnel and model work end to end before wiring up a client.
  </Step>

  <Step title="Create an API key">
    In the [Keys](https://tokios.com/console/keys) tab, click **Create key**. Give it a display name and, optionally, model patterns to scope it (leave `*` to allow all your models). Copy the `sk-tok-…` key immediately — it is shown only once.
  </Step>

  <Step title="Point your client and test">
    Use the base URL that matches your client:

    - **OpenAI style** (Codex, OpenAI SDKs): `https://api.tokios.com/v1`
    - **Anthropic style** (Claude Code): `https://api.tokios.com`

    Then send a request. Replace the key and model with your own:

    ```bash
    curl https://api.tokios.com/v1/chat/completions \
      -H "Authorization: Bearer sk-tok-YOUR_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gemma-tunnel",
        "messages": [{"role": "user", "content": "Hello!"}]
      }'
    ```

    A successful call returns an OpenAI-shaped completion (abbreviated):

    ```json
    {
      "model": "gemma-tunnel",
      "choices": [
        {
          "message": { "role": "assistant", "content": "Hello! How can I help?" },
          "finish_reason": "stop"
        }
      ]
    }
    ```

    <Tip>
      Prefer not to use curl? You can run the same test from the [Playground](https://tokios.com/console/playground) — pick your model, enter a prompt, and click **Run**.
    </Tip>
  </Step>
</Steps>
