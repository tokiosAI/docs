title: "Get Started with Tokios in Under 5 Minutes"
sidebarTitle: "Quickstart"

---

Tokios gets your local model online in a few steps: create a connector, run it next to your model, register the model, and call it. No port forwarding, no public IP, no NAT configuration. The connector dials out to Tokios once, and from that moment your model is reachable at a stable cloud endpoint that any OpenAI- or Anthropic-compatible client can target.

<Steps>
  <Step title="Create your account">
    Sign up at [tokios.com](https://tokios.com) — it's free and no credit card is required. After signing up you'll land in the Tokios dashboard, where you manage connectors, registered models, and API keys.
  </Step>
  <Step title="Create your account">
    Sign up at [tokios.com](https://tokios.com) — it's free and no credit card is required. After signing up you'll land in the Tokios dashboard, where you manage connectors, models, and API keys.
  </Step>
  <Step title="Create a connector">
    In the dashboard, create a **connector**. Tokios issues a `ConnectorId` and a one-time `Token` — copy the token now; it's shown only once.
  </Step>
  <Step title="Download and run the connector">
    Grab the `tokios-connector` binary for your platform from the dashboard's **Downloads** page.

    | Platform | Download |
    | --- | --- |
    | Windows x64 | `tokios-connector.exe` |
    | macOS arm64 | `tokios-connector` |
    | Linux x64 | `tokios-connector` |

    Create a minimal `tokios.json` config file next to the binary, replacing `<your-tunnel-token>` with the tunnel token shown in the dashboard:

    ```json tokios.json
    {
      "tunnel_token": "<your-tunnel-token>",
      "upstream": "http://localhost:11434"
    }
    ```

    <Note>
      `11434` is Ollama's default port. If you're running llama.cpp, vLLM, or LM Studio, update `upstream` to match their listening address (e.g. `http://localhost:8080`).
    </Note>

    Then start the connector:

    <CodeGroup>

    ```bash macOS / Linux
    ./tokios-connector --config tokios.json
    ```

    ```powershell Windows
    tokios-connector.exe --config tokios.json
    ```

    </CodeGroup>

    You should see a confirmation line like `[tokios] tunnel connected` in your terminal. The connector is now holding an outbound WebSocket to `api.tokios.com` — no inbound ports were opened on your machine.
  </Step>
  <Step title="Run the connector next to your model">
    Download the `tokios-connector` binary for your platform from the dashboard's **Downloads** page, then create a `tokios-connector.json` beside it:

    ```json tokios-connector.json
    {
      "Connector": {
        "ConnectorId": "con_your_connector_id",
        "Token": "ct-your-connector-token",
        "Models": ["gemma4:e4b"],
        "Upstream": { "BaseUrl": "http://127.0.0.1:11434/v1" }
      }
    }
    ```

    `Models` is the model id your local server uses, and `BaseUrl` points at it (the default is Ollama — see [Backends](/setup/supported-backends)). Start the connector:

    <CodeGroup>

    ```bash macOS / Linux
    ./tokios-connector --config tokios-connector.json
    ```

    ```powershell Windows
    tokios-connector.exe --config tokios-connector.json
    ```

    </CodeGroup>

    The connector opens a single outbound WebSocket to Tokios — no inbound ports.
  </Step>

  <Step title="Register your model">
    In the Tokios dashboard, navigate to **Connectors** and click **Pair Connector**. Select the connector that just came online and give your model a name — for example, `gemma-tunnel`.

    Tokios registers the model and assigns it a stable, cloud-reachable deployment identifier tied to your account. You'll use this name as the `model` field in every API request.
  </Step>
  <Step title="Register your model">
    Back in the dashboard, create a **deployment**: give it a public model name — for example, `gemma-tunnel` — select the connector you just started, and point it at the upstream model id the connector serves (`gemma4:e4b`). This public name is what you send as the `"model"` field in every API request.
  </Step>

This public name is what you send as the `"model"` field in every API request.

  <Step title="Mint an API key">
    Go to **API Keys** in the dashboard and click **Generate Key**. Tokios issues a tenant-scoped key in the form `sk-tokios-...`.

    Copy it now and store it somewhere safe (a password manager or secrets vault). The dashboard will not show it again after you close the dialog.
  </Step>

  </Step>
  <Step title="Mint an API key">
    Go to **API Keys** and click **Generate Key**. Tokios issues a tenant-scoped key in the form `sk-tokios-...`. Copy it now and store it somewhere safe — the dashboard won't show it again.
  </Step>
  <Step title="Make your first call">
    With the connector running and your key in hand, send a request to the Tokios endpoint. Swap in your real key and model name:

    ```bash
    curl https://api.tokios.com/v1/chat/completions \
      -H "Authorization: Bearer sk-tokios-YOUR_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gemma-tunnel",
        "messages": [{"role": "user", "content": "Hello!"}]
      }'
    ```

    A successful response looks like this:

    ```json
    {
      "id": "chatcmpl-a1b2c3d4",
      "object": "chat.completion",
      "created": 1718000000,
      "model": "gemma-tunnel",
      "choices": [
        {
          "index": 0,
          "message": {
            "role": "assistant",
            "content": "Hello! How can I help you today?"
          },
          "finish_reason": "stop"
        }
      ],
      "usage": {
        "prompt_tokens": 10,
        "completion_tokens": 9,
        "total_tokens": 19
      }
    }
    ```

    Your local model just answered a call from the cloud — without ever opening a port. 🎉
  </Step>
