---
title: "Get started with Tokios in under 5 minutes"
sidebarTitle: "Quickstart"
description: "Install the Tokios Connector, pair your local model, and make your first OpenAI-compatible API call in under five minutes without port forwarding."
---

Tokios gets your local model online in a few steps. Create a connector, run it next to your model, register a deployment, and call it from any OpenAI-compatible client.

You do not need port forwarding, a public IP, or NAT configuration. The connector dials out to Tokios once, then your model is reachable through a stable cloud endpoint.

<Steps>
  <Step title="Create your account">
    Sign up at [tokios.com](https://tokios.com). No credit card is required.

    After signing up, you land in the Tokios dashboard. Use the dashboard to manage connectors, deployments, and API keys.
  </Step>

  <Step title="Create a connector">
    In the dashboard, create a **connector**. Tokios issues a `ConnectorId` and a one-time `Token`.

    Copy the token now. The dashboard shows it only once.
  </Step>

  <Step title="Run the connector next to your model">
    Download the `tokios-connector` binary for your platform from the dashboard's **Downloads** page.

    Create `tokios-connector.json` next to the binary:

    ```json tokios-connector.json
    {
      "Connector": {
        "ConnectorId": "con_your_connector_id",
        "Token": "ct-your-connector-token",
        "Models": ["gemma4:e4b"],
        "Upstream": {
          "BaseUrl": "http://127.0.0.1:11434/v1"
        }
      }
    }
    ```

    `Models` is the upstream model id your local server uses. `BaseUrl` points at its OpenAI-compatible endpoint.

    <Note>
      Ollama uses `http://127.0.0.1:11434/v1` by default. For llama.cpp, vLLM, or LM Studio, use the matching URL from [Backends](/setup/supported-backends).
    </Note>

    Start the connector:

    <CodeGroup>

    ```bash macOS / Linux
    ./tokios-connector --config tokios-connector.json
    ```

    ```powershell Windows
    tokios-connector.exe --config tokios-connector.json
    ```

    </CodeGroup>

    The connector opens a single outbound WebSocket to Tokios. It does not open inbound ports on your machine.
  </Step>

  <Step title="Register your deployment">
    Back in the dashboard, create a **deployment**.

    Give it a public model name, for example `gemma-tunnel`. Select the connector you just started, then point it at the upstream model id the connector serves, such as `gemma4:e4b`.

    Use the public deployment name as the `model` field in API requests.
  </Step>

  <Step title="Create an API key">
    Go to **API Keys** and click **Generate Key**. Tokios issues a tenant-scoped key in the form `sk-tokios-...`.

    Copy the key now and store it in a password manager or secrets vault. The dashboard will not show it again.
  </Step>

  <Step title="Make your first call">
    With the connector running and your key in hand, send a request.

    Replace `sk-tokios-YOUR_KEY` and `gemma-tunnel` with your real API key and public deployment name:

    ```bash
    curl https://api.tokios.com/v1/chat/completions \
      -H "Authorization: Bearer sk-tokios-YOUR_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gemma-tunnel",
        "messages": [{"role": "user", "content": "Hello!"}]
      }'
    ```

    A successful response looks like this:

    ```json
    {
      "id": "chatcmpl-a1b2c3d4",
      "object": "chat.completion",
      "created": 1718000000,
      "model": "gemma-tunnel",
      "choices": [
        {
          "index": 0,
          "message": {
            "role": "assistant",
            "content": "Hello! How can I help you today?"
          },
          "finish_reason": "stop"
        }
      ],
      "usage": {
        "prompt_tokens": 10,
        "completion_tokens": 9,
        "total_tokens": 19
      }
    }
    ```
  </Step>
</Steps>
