title: "Register a Model Deployment in the Tokios Dashboard"
sidebarTitle: "Register Model"

---

A **deployment** maps a public model name to a connector and the upstream model id it serves. The public name is the string you pass as the `"model"` field in every API call — for example, `gemma-tunnel`.

## Register your deployment

<Steps>
  <Step title="Open the dashboard">
    Go to your [Tokios account dashboard](https://tokios.com/dashboard). When the connector is running and the tunnel is live, you'll see its status displayed as **Connected**.

    If the status shows **Waiting** or **Disconnected**, check that the connector is running on your local machine and that `tokios.json` contains the correct `tunnel_token`.
  </Step>
  <Step title="Confirm your connector is online">
    Open the [Tokios dashboard](https://tokios.com/dashboard). With the connector running, it shows as **Connected**. If it shows **Waiting** or **Disconnected**, check that the connector is running and that `tokios-connector.json` has the correct `ConnectorId` and `Token`.
  </Step>
  <Step title="Pair the connector">
    Click **Pair Connector**. The dashboard will either:

    - **Auto-detect** the running connector and prompt you to confirm the pairing, or
    - Ask you to **enter the pairing code** shown in the connector's terminal output.

    The pairing code is displayed in the connector's startup logs, directly below the `Tunnel established` message.
  </Step>
  <Step title="Name the deployment">
    Choose a model name for this deployment — for example, `gemma-tunnel` or `llama3-local`. This is the exact string you will pass as the `"model"` field in every API request routed to this connector.

    **Naming rules:**

    - Lowercase alphanumeric characters only
    - Hyphens (`-`) are allowed
    - No spaces, underscores, or special characters

    Good examples: `gemma-tunnel`, `llama3-local`, `mistral-home`
  </Step>
  <Step title="Create a deployment">
    Create a new deployment and fill in:

    - **Public name** — the model name clients will use, e.g. `gemma-tunnel`.
    - **Connector** — the connector that serves this model.
    - **Upstream model** — the model id the connector forwards to your local server (what your server calls it, e.g. `gemma4:e4b`).
  </Step>
  <Step title="Save">
    Click **Register**. The model now appears as an active deployment under your account and is immediately reachable at `api.tokios.com`.
  </Step>
  <Step title="Save">
    Save the deployment. The model is immediately reachable at `api.tokios.com` under its public name.
  </Step>
</Steps>

**Public name rules:**

- Lowercase alphanumeric characters and hyphens (`-`)
- No spaces, underscores, or special characters

Good examples: `gemma-tunnel`, `llama3-local`, `mistral-home`.

## Use the public name in API calls

Pass the public name as the `model` field. The gateway resolves it to your connector and rewrites it to the upstream model id before forwarding:

```json
{
  "model": "gemma-tunnel",
  "messages": [{ "role": "user", "content": "What is the capital of France?" }]
}
```

<Note>
  One connector can serve several models — register a deployment for each public name, all pointing at the same connector with different upstream model ids. See [Model Routing](/guides/model-routing).
</Note>
